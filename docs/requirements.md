## 1. プロジェクト概要

### 1.1 目的

クラウド型勤怠管理SaaSの要件を定義するドキュメントです。多様な規模・業種の企業がすぐに利用できる、拡張性の高い勤怠管理プラットフォームの構築を目指します。

### 1.2 サービスコンセプト

- マルチテナント対応のSaaSとして外部企業に提供
- 企業規模・業種を問わず導入可能な柔軟性
- 法令遵守（労働基準法対応）を標準サポート
- モバイル・PCの双方からアクセス可能

### 1.3 ロール定義

| ロール | 説明 | 主な操作 |
|---|---|---|
| スーパー管理者 | サービス提供側（自社）の管理者 | テナント管理、課金管理、全体監視 |
| テナント管理者 | 導入企業の管理者・HR担当者 | 従業員管理、勤怠確認・承認、レポート出力 |
| マネージャー | チームリーダー・部門管理者 | 担当チームの勤怠確認・承認、シフト作成 |
| 一般ユーザー | 従業員 | 打刻、シフト確認、申請 |

## 2. 機能要件

### 2.1 出退勤打刻機能

#### 打刻方法

| 打刻方法 | 概要 | 優先度 |
|---|---|---|
| Webブラウザ打刻 | PCやスマートフォンのブラウザから打刻 | 必須 |
| GPS打刻 | スマートフォンのGPS情報と合わせて打刻（位置確認） | 推奨 |
| QRコード打刻 | 共用端末にQRコードを表示し、従業員がスキャンして打刻 | 推奨 |
| ICカード打刻 | 交通系ICカードやIDカードによる打刻（専用リーダー連携） | 将来対応 |

#### 打刻の種類

- 出勤 / 退勤
- 休憩開始 / 休憩終了
- 直行（外出先からの出勤）/ 直帰（外出先からの退勤）

#### 打刻補正・修正

- 管理者・マネージャーによる打刻時刻の修正機能
- 修正履歴の記録（誰がいつ修正したか）
- 本人申請による打刻修正ワークフロー（申請→承認）

---

### 2.2 シフト管理機能

#### シフトパターン定義

| 機能 | 内容 |
|---|---|
| シフトパターン登録 | 「早番（8:00-17:00）」「遅番（13:00-22:00）」等の名前付きパターンを事前登録 |
| カスタムシフト | 都度任意の時間帯を設定可能 |
| 休日パターン | 定休日・会社カレンダー（祝日・特定休日）の設定 |
| フレックスタイム対応 | コアタイム設定、フレキシブルタイムの範囲設定 |

#### シフト作成・配布

- 月次・週次のシフト表を管理者/マネージャーが作成
- ドラッグ&ドロップによる直感的なシフト編集UI
- シフト公開・通知（メール・アプリ内通知）
- シフト希望の収集機能（従業員が希望日時を事前登録）
- 希望を考慮した自動シフト案の生成（AIアシスト：将来対応）

#### シフト変更管理

- シフト変更申請ワークフロー
- 代替要員の検索・調整支援
- 変更履歴の記録

---

### 2.3 申請・ワークフロー機能

| 申請種別 | 内容 | 承認フロー |
|---|---|---|
| 残業申請 | 事前申請または事後申請（上限時間警告付き） | 直属マネージャー→管理者 |
| 有給休暇申請 | 有給残数の自動管理、法定有給の付与管理 | 直属マネージャー |
| 特別休暇 | 慶弔・育児・介護等 | 管理者 |
| 振替休日 | 休日出勤に対する振替日申請 | 直属マネージャー |
| 打刻修正申請 | 自身の打刻ミスの修正依頼 | 直属マネージャー |
| 在宅勤務申請 | リモートワーク日の申請・記録 | 直属マネージャー |

**承認フロー設定**

- 最大3段階の承認ルートを設定可能
- 代理承認機能（承認者不在時）
- 承認期限・督促通知

### 2.4 管理者向けレポート機能

| レポート名 | 内容 | 出力形式 |
|---|---|---|
| 月次勤怠サマリー | 従業員別の勤務時間・残業時間・欠勤日数の集計 | 画面/CSV/PDF |
| 残業時間レポート | 36協定の上限管理、アラート表示 | 画面/CSV |
| 有給消化レポート | 有給取得状況・残日数・法定取得率 | 画面/CSV/PDF |
| 出勤率レポート | 部署別・期間別の出勤率 | 画面/グラフ/CSV |
| 打刻状況レポート | 未打刻・異常打刻の一覧 | 画面/CSV |
| コスト分析レポート | 労働コストの部署別・役職別集計（給与連携時） | 画面/CSV |

**カスタムレポート**

- 任意の期間・部署・条件でのデータ抽出
- 項目選択による柔軟なCSVエクスポート
- 定期レポートの自動メール配信設定

## 3. 差別化機能 — ワークタイプ連動打刻・精算 ★

SES・派遣・受託など「複数の業務形態・契約単位で従業員が稼働する」業種向けの差別化機能です。

### 3.1 背景・課題

| 課題 | 具体例 |
|---|---|
| 稼働区分の混在 | 同日に原籍業務2h＋客先A現場4hを行った場合、単純な出退勤打刻では区別不可 |
| 請求計算の手動対応 | 客先ごとの稼働時間を手動でExcel集計しており、ミス・工数が発生 |
| 原価の不可視化 | 自社業務と外部業務のコスト配賦が曖昧になりやすい |
| 多契約の並行管理 | 複数の客先に同時アサインされている場合の管理が煩雑 |

### 3.2 ワークタイプ定義（SES企業の設定例）

| ID | 種別名 | 説明 | 精算単位 |
|---|---|---|---|
| WT-01 | 原籍業務 | 自社オフィスでの社内業務 | 月次・社内原価 |
| WT-02 | 現場業務（客先A） | 株式会社○○への常駐・派遣業務 | 月次・客先請求 |
| WT-03 | 現場業務（客先B） | △△社へのリモート支援業務 | 月次・客先請求 |
| WT-04 | 研修・教育 | 社内研修・資格取得対応 | 月次・社内原価 |
| WT-05 | 移動時間 | 客先往復の移動（契約により請求対象外） | 参考値 |

### 3.3 精算モデル

| 精算方式 | 内容 |
|---|---|
| 完全固定（fixed） | 稼働時間に関わらず月額固定 |
| 時間精算（hourly） | 稼働時間 × 時間単価で毎月変動 |
| 上下幅あり（range） | min_hours〜max_hours の範囲内は固定、超過・不足は単価×時間で精算 |
| プロジェクト（project） | 成果物・マイルストーンベースの精算 |

## 4. 非機能要件

| カテゴリ | 要件 | 目標値・基準 |
|---|---|---|
| パフォーマンス | ページ読み込み速度 | 3秒以内（通常操作） |
| パフォーマンス | 打刻レスポンス | 1秒以内 |
| 可用性 | 稼働率 | 99.9%以上（月次SLA） |
| セキュリティ | データ暗号化 | 通信：TLS1.3 / 保存：AES-256 |
| セキュリティ | ログ保管 | 操作ログ5年保管（法令準拠） |
| 法令対応 | 労働基準法 | 36協定・有給管理の法定要件を標準対応 |
