---

## プロジェクト概要

SES業界向けクラウド型勤怠管理SaaSです。
**最大の差別化機能：ワークタイプ（作業種別）連動の打刻・精算**
一人の従業員が原籍業務・客先A・客先Bなど複数種別をまたいで稼働でき、
種別ごとに稼働時間を集計して客先への請求を自動計算します。

## 絶対に守るルール（全ファイル共通）

1. `any` 型を使わない
2. `app.*` テーブルへのクエリには必ず `WHERE tenant_id = tenantId` を入れる（RLSは保険）
3. `audit_logs` への `UPDATE` / `DELETE` は禁止
4. 複数テーブルへの書き込みは必ずトランザクションで行う
5. `console.log` を本番コードに残さない

## ローカル開発の起動

```bash
yarn install
docker compose up -d      # PostgreSQL + Redis
yarn workspace api db:migrate
yarn dev                  # 全アプリ同時起動
```
